## 1. Business purpose

### 1) Project background
음악은 현대 사회에서 빠져서는 안 될 필수적인 요소 중 하나로 자리매김하고 있다. 우리 주변을 둘러보면 길거리, 카페, 상점 등 다양한 장소에서 음악이 흐르고 있는 것을 볼 수 있을 것이다. 데이터플랫폼 기업 아이지에이웍스 모바일인덱스에 따르면 25년 6월 기준, 유튜브와 유튜브 뮤직 교차사용자 규모가 약 712만명으로 추산됐다. 매년 이용자 수가 증가하는 추세이고 이를 통해 음악은 우리의 일상에 매우 밀접하게 다가와 있다는 것을 알 수 있다.

음악은 댄스, 팝, 발라드, R&B, 인디 등 다양한 장르가 있으며, 각각의 곡은 특유의 스타일과 분위기를 지니고 있다. 사람들은 음악을 들을 때, 날씨나 개인의 기분에 따라 어떤 스타일의 음악이 끌리는지 날마다 다를 수 있다. 즉, 음악은 우리의 감정과 상황을 표현하고 공유하는 객관적인 수단이 될 수 있다. 

하지만 매번 내 기분에 맞는 곡을 찾기란 쉽지 않다. 때로는 시간이 부족하거나 음악의 다양성에 적응하기 어려울 수 있다. 기존 거대 플랫폼의 추천 알고리즘은 단순히 과거의 청취 이력이나 대중적인 인기도에 의존하기 때문에, 당장의 감정이나 상황을 즉각적으로 반영해주기는 어렵다. 따라서 사용자가 능동적으로 자신의 상황과 감정에 맞는 음악을 선택하고, 직관적인 플레이리스트로 분류하여 관리할 수 있는 개인화된 큐레이션 시스템의 필요성이 대두되었다.

### 2) Goal
'My Mood, My Music' 이하 M3는 사용자가 직접 유튜브 영상 링크를 기반으로 태그를 부여하여, 클릭 한 번으로 그날의 무드에 맞는 음악을 필터링하고 감상할 수 있게 해주는 **맞춤형 음악 큐레이션 웹 서비스**를 개발하는 것을 목표로 한다. 이 애플리케이션의 알고리즘은 사용자의 선호도와 기분, 그리고 주변 환경을 고려하여 맞춤형 추천을 제공한다. 나아가, 새로운 음악을 발견하고 다양한 스타일을 경험함으로써 음악적 호기심을 키우는 데도 도움이 될 것이다.

### 3) Target Market
* **사용층 :** 음악 감상이 일상에 스며든 사람이거나 스며들고 싶은 사람들. 감정이나 날씨에 맞춰 수동적으로 플레이리스트를 만들어 듣는 사람들. 그날의 날씨나 감정에 맞춰 능동적으로 플레이리스트를 디깅하고 소장하려는 학생 및 직장인.

## 2. System context diagram

##### 사용자 <-> 시스템 <-> 관리자
* **Login/Logout :** 로그인 및 로그아웃
* **Join :** 회원가입

##### 사용자 <-> 시스템
* **Mypage :** 마이페이지
* **Register Music :** 음악 등록 
* **Filter by Mood Tag :** 태그별 음악 필터링
* **View My Playlist :** 내 플레이리스트 관리
* **View Social Playlist :** 타 유저 플레이리스트 조회 요청
* **Like Playlist :** 타 유저 플레이리스트 좋아요(소셜기능)
* **Confirm Music Saved :** 음악 등록 성공 알림
* **Give Playlist Data :** 요청한 플레이리스트 데이터 전송

##### 시스템 <-> 관리자
* **View System Statistics :** 시스템 통계 조회
* **Register Official Music :** 샘플 음악 등록

##### 시스템 <-> 외부 API
* **Request Video Data :** 유튜브 영상 데이터 요청
* **Give Video Data :** 영상 제목 및 고화질 썸네일 전송

## 4. Concept of operation

**1) Login / Logout**

| Category | Description |
| :--- | :--- |
| Purpose | 사이트 내 기능을 이용하기 위해 등록된 사용자인지 확인 후 접속 및 종료하기 위함이다. |
| Approach | 아이디와 비밀번호를 입력받아 DB 정보와 대조하여 로그인 여부를 결정하며, 로그아웃 버튼을 누르면 세션을 종료하도록 설계한다. |
| Dynamics | 해당 서비스를 이용하거나 종료할 경우 |
| Goals | 사용자 인증을 통해 권한에 맞는 안전한 개인화 시스템 환경을 제공한다. |

**2) Join**

| Category | Description |
| :--- | :--- |
| Purpose | 회원이 되어 나만의 플레이리스트를 구축하고 관리하기 위함이다. |
| Approach | 회원가입 화면에서 아이디, 비밀번호, 닉네임을 입력하여 서버에 전송하면 데이터베이스 User 테이블에 계정 정보가 등록된다. |
| Dynamics | 서비스를 처음 이용하는 신규 사용자(또는 최초 관리자)인 경우 |
| Goals | 이용자의 계정 정보를 안전하게 시스템에 저장하고 가입 절차를 완료한다. |

**3) Mypage**

| Category | Description |
| :--- | :--- |
| Purpose | 이용자의 개인 정보 관리 및 자신이 등록한 음악들을 한곳에서 확인하기 위함이다. |
| Approach | 마이페이지 메뉴에 진입하여 닉네임을 변경할 수 있는 폼을 제공하고, 하단에는 본인의 계정으로 등록된 노래들을 렌더링한다. |
| Dynamics | 개인 정보를 수정하거나, 자신이 등록한 곡들을 모아보고자 할 경우 |
| Goals | 개인정보 수정 기능 및 자신만의 플레이리스트 열람을 제공한다. |

**4) Register Music**

| Category | Description |
| :--- | :--- |
| Purpose | 선호하는 유튜브 음악을 태그 기반 플레이리스트에 등록하기 위함이다. |
| Approach | 유튜브 영상의 11자리 Video ID와 원하는 태그를 고르거나 입력하고, 저장 버튼을 눌러 시스템에 등록을 요청한다. |
| Dynamics | 마음에 드는 유튜브 음악을 자신의 리스트에 등록하거나 타인에게 추천하고 싶을 경우 |
| Goals | 입력된 ID와 태그 정보를 바탕으로 음악 데이터를 시스템 데이터베이스에 추가한다. |

**5) Filter by Mood Tag**

| Category | Description |
| :--- | :--- |
| Purpose | 이용자가 현재 원하는 분위기의 음악만을 빠르고 직관적으로 찾기 위함이다. |
| Approach | 화면 상단에 나열된 태그 버튼을 클릭하면, 해당 태그와 일치하는 음악만 필터링하여 보여준다. |
| Dynamics | 날씨, 기분, 상황 등에 맞춘 분위기의 곡들로만 구성된 플레이리스트를 듣고 싶을 경우 |
| Goals | 별도의 폴더 생성 과정 없이, 태그 필터링을 통해 자동화된 맞춤형 플레이리스트를 제공한다. |

**6) View My Playlist**

| Category | Description |
| :--- | :--- |
| Purpose | 이용자가 직접 등록한 음악들을 관리하여 업데이트를 유지하기 위함이다. |
| Approach | 본인이 등록한 음악을 조회하거나, 개별 곡의 태그를 다른 무드로 변경 또는 삭제 버튼을 눌러 리스트를 정리한다. |
| Dynamics |노래를 삭제하거나, 분위기에 더 어울리는 태그로 변경하고 싶을 경우 |
| Goals | 본인이 등록한 음악 데이터에 대한 관리 권한을 부여한다. |

**7) View Social Playlist**

| Category | Description |
| :--- | :--- |
| Purpose | 다른 이용자의 플레이리스트를 보며 새로운 음악을 공유받기 위함이다. |
| Approach | 둘러보기 메뉴에서 타 유저의 닉네임을 클릭하면, 해당 유저가 등록했던 태그별 플레이리스트 화면을 열람 전용으로 조회한다. |
| Dynamics | 다른 사람들은 어떤 음악을 듣는지 궁금하거나 새로운 곡을 추천받고 싶을 경우 |
| Goals | 다른 이용자의 플레이리스트를 조회하는 소셜 큐레이션 경험을 제공한다. |

**8) Like Playlist**

| Category | Description |
| :--- | :--- |
| Purpose | 다른 이용자가 등록한 음악에 긍정적인 피드백을 남기기 위함이다. |
| Approach | 타 유저의 플레이리스트를 열람하던 중, 마음에 드는 개별 음악 카드의 '좋아요' 버튼을 클릭하여 시스템에 상호작용 기록을 남긴다. |
| Dynamics | 남의 리스트에서 내 취향에 딱 맞는 좋은 노래를 발견했을 경우 |
| Goals | 직관적인 '좋아요' 기능을 통해 사용자 간의 소셜 네트워크 및 커뮤니티 형성을 유도한다. |

**9) Confirm Music Saved**

| Category | Description |
| :--- | :--- |
| Purpose | 음악 등록 요청이 정상적으로 완료되었음을 이용자에게 시각적으로 인지시키기 위함이다. |
| Approach | 백엔드에서 DB 저장이 성공적으로 끝나면 프론트엔드로 성공 응답 코드를 전송하고, 화면에 "저장 성공!" 알림창을 띄워준다. |
| Dynamics | 이용자가 새로운 음악을 시스템에 등록 요청한 직후 |
| Goals | 신속하고 정확한 피드백을 통해 시스템 프로세스의 신뢰성을 높인다. |

**10) Give Playlist Data**

| Category | Description |
| :--- | :--- |
| Purpose | 이용자가 요청에 맞는 정확한 음악 리스트를 출력하기 위함이다. |
| Approach | 시스템이 필터링 조건에 맞춰 DB에서 추출한 결과를 JSON 형태의 데이터로 클라이언트에 전송한다. |
| Dynamics | 페이지를 이동하거나 새로고침하여 최신 음악 목록을 불러와야 할 경우 |
| Goals | 조건에 부합하는 플레이리스트 데이터를 클라이언트에 제공한다. |

**11) Request Video Data**

| Category | Description |
| :--- | :--- |
| Purpose | 이용자가 입력한 Video ID만으로는 알 수 없는 실제 영상의 썸네일과 제목 정보를 획득하기 위함이다. |
| Approach | 시스템이 유튜브 Data API v3 서버로 해당 Video ID를 포함한 GET 방식의 데이터 제공 요청을 보낸다. |
| Dynamics | 이용자가 새로운 음악 등록을 요청했을 때 내부 로직으로 자동 발생 |
| Goals | 외부 API와의 원활한 통신을 통해 음악 메타데이터 확보를 시도한다. |

**12) Give Video Data**

| Category | Description |
| :--- | :--- |
| Purpose | 시스템이 요청한 영상이 실제로 존재하는지 확인하고, 화면 구성에 필요한 메타데이터를 넘겨주기 위함이다. |
| Approach | 유튜브 API 서버가 요청받은 ID를 검증한 후, 이상이 없다면 해당 영상의 제목과 고화질 썸네일 이미지 주소를 시스템으로 반환한다. |
| Dynamics | 유튜브 API가 시스템으로부터 정상적인 데이터 요청을 수신했을 때 |
| Goals | 공식적인 유튜브 데이터 전송을 통해 시스템 내 음원 및 영상 데이터의 무결성을 보장한다. |

**13) View System Statistics**

| Category | Description |
| :--- | :--- |
| Purpose | 관리자가 서비스의 활성도와 규모를 한눈에 파악하고 시스템 서버의 과부하 여부를 점검하기 위함이다. |
| Approach | 관리자 권한을 가진 계정으로 로그인 시, 시스템이 DB의 총 회원 수와 등록된 음악 개수를 카운트하여 간단한 대시보드 화면에 출력한다. |
| Dynamics | 관리자가 시스템의 전반적인 운영 현황 및 트래픽을 점검하고자 할 경우 |
| Goals | 시스템의 현재 상태를 나타내는 직관적인 통계 데이터를 관리자에게 제공한다. |

**14) Register Official Music**

| Category | Description |
| :--- | :--- |
| Purpose | 서비스 초기 이용자가 아무도 없어 화면이 텅 비어 보이는 것을 방지하기 위함이다. |
| Approach | 관리자가 직접 양질의 샘플 곡들을 분위기별 태그로 다수 등록하여, 로그인을 하지 않은 이용자들의 홈 화면에도 공식 추천 리스트가 노출되도록 한다. |
| Dynamics | 시스템 론칭 초기나, 특정 시즌에 맞춰 공식 추천 리스트를 배포할 경우 |
| Goals | 관리자가 능동적으로 샘플 음악을 등록하여 서비스 초기 품질을 확보하고 이용자 참여를 유도한다. |

## 5. Problem statement 

**1) DB 영속성 문제**
* **문제:** 개발 초기 단계에서 H2 Database를 기본으로 두고 작업을 진행하였다. 그러다 보니 코드를 수정하거나 서버를 재시작할 때마다 등록해둔 데이터들이 날아가는 문제점이 발생했다.
* **해결 방안:** 데이터베이스 연결 설정을 로컬의 물리적 파일 기반으로 변경하여, 서버가 종료되어도 데이터가 안전하게 유지되도록 영속성을 확보할 것이다.

**2) 실시간 렌더링 지연 업로드 누락 문제**
* **문제:** 이용자가 유튜브 링크를 가져와 새로운 음악을 등록했을 때, DB 상에는 정상적으로 등록이 되지만 사용자가 보고 있는 화면에서는 새로고침을 해야만 노래가 뜨는 현상이 발생할 수 있다.
* **해결 방안:** 프론트엔드에서 `axios.post` 성공 응답을 받은 직후 `fetchMusicList()`를 즉시 재호출하여, 새로고침 없이도 즉각적으로 화면에 추가된 음악이 반영되도록 로직을 수정한다.

**3) 외부 API 연동 시 보안 Profile 설정 오류**
* **문제:** YouTube API 키를 메인 설정 파일에서 해당 시크릿 파일을 불러오는 문법을 오기입하여, 백엔드에 실제 키가 아닌 문자열 자체를 전송하게 되었고 500 Internal Server Error가 발생한다.
* **해결 방안:** 콘솔 에러 로그를 분석하여 API 인증 실패가 원인임을 파악한다. 메인 설정 파일의 코드를 올바르게 수정하여, Spring Boot 구동 시 정상적으로 병합되도록 조치한다.

**NFRs 비기능적 요구사항**
1. 애플리케이션은 빠르고 신속한 응답 시간을 제공해야한다. 
2. 이용자 수가 증가해도 시스템이 확장 가능해야한다. 
3. 이용자의 개인정보 및 데이터 보호를 위한 보안 기능이 강화되어야한다.
4. 이용자 인터페이스가 직관적이고 사용하기 쉬워야한다.
5. 서비스 중단 시간을 최소화하고 복구 시간을 최적화해야한다.
6. 음악 저작권에 관련된 법을 엄격하게 준수해야한다.

## 6. Glossary

* **알고리즘 :** 어떤 문제를 해결하기 위하여, 입력된 자료를 바탕으로 원하는 출력을 유도하는 규칙의 집합. 
* **큐레이션 :** 콘텐츠를 목적에 따라 분류하고 배포하는 일.
* **디깅 :** 좋아하는 특정 품목이나 영역에 깊게 파고들며 적극적으로 소비하는 행위.
* **클라이언트 :** 정보 통신망에서, 네트워크를 통해 서버 시스템에 연결되어 정보나 서비스를 제공받는 컴퓨터 또는 소프트웨어.
* **썸네일 :** 페이지 전체의 레이아웃을 검토할 수 있게 페이지 전체를 작게 줄여 화면에 띄운 것 또는 축소된 미리보기 이미지.
* **렌더링 :** 평면인 그림에 외부의 정보에 따라 실감나는 화상을 만들어 내거나, 2차원의 화상에 입체감을 불어넣어 화면에 띄워주는 과정.
* **필터링 :** 데이터 통신이나 검색에서, 이미 알려진 특정 기준에 따라 자료, 신호, 정보 따위를 가려내는 과정.
* **프로세스 :** 컴퓨터 시스템 내에서 연속적으로 실행되고 있는 프로그램.
* **로직 :** 컴퓨터 프로그램 내에서 데이터가 처리되거나 기능이 작동하는 논리적인 흐름이나 구조.
* **메타데이터 :** 대량의 정보 가운데에서 확인하고자 하는 정보를 효율적으로 검색하기 위해 특정한 규칙을 가지고 부여되는 부가적인 데이터.
* **무결성 :** 데이터베이스 내에 저장된 데이터의 정확성, 일관성, 유효성이 결함이나 오류 없이 유지되는 성질.
